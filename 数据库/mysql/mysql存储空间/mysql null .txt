一） ：定义字段类型的时候，如果允许值为NULL，那么就要为每条数据多分配一个字节的空间来表示字段是否为空，
无论报错的值是否为空都要占用这一字节。理解上面的内容后，可以回答你NULL要占用的空间

2）：mysql null 列可以使用索引并且is null 也可以使用到索引

3）： null 对索引的影响

MySQL难以优化引用了可空列的查询，【它会使索引、索引统计】和值更加复杂。
可空列需要更多的储存空间，还需要在MySQL内部进行特殊处理。当可空列被索引的时候，
每条记录都需要一个额外的字节【保存字段是否为null】，还可能导致 MyISAM 中固定大小的索引(例如一个整数列上的索引)变成可变大小的索引
总结三点： 
1：count distinct 等计算函数数据丢失；
2：索引会占用更多的存储空间  
3：使用困难 select * from person where name<>null; 无数据，查询失效，其实是存在数据的
4： varchar类型索引可以 like ,=，> 等使用 

4）： null存在意义： 可以节省空间 比如int 类型的默认为空占用4 字节，但是null 只占用1byte
